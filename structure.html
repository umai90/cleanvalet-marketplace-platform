Platform Model

Two-Sided Marketplace

CleanValet connects two distinct user groups — customers who need cleaning, and providers who do the cleaning — with the platform earning commission in the middle.

👤 Customer Side

  • Browse services and provider listings
  • Book pickup via the hero form widget
  • Enter postcode, select service, pick date
  • Track order through 5 live stages
  • Pay zero platform booking fees
  • Receive cleaned clothes within 24 hours
  • Leave rating and review for provider

⚡ CleanValet Platform

  • Routes orders to providers
  • Handles all payments
  • Vets and approves providers
  • Earns 12–18% commission
  • Processes weekly payouts
15% avg. commission
⇄

🏪 Provider Side

  • Apply to join via provider form
  • Admin reviews and approves account
  • Receive orders through the platform
  • Accept or decline any order (flexible)
  • Perform professional cleaning service
  • Keep 82–88% of every order value
  • Get paid every Friday automatically

JavaScript

All Interactive Features

Every JavaScript function across both files — pure vanilla JS, zero external libraries.

index.html — Core UI

6 features
scroll event
Navbar transitions from transparent to solid navy with backdrop-blur after 60px scroll
hamburger()
Toggles mobile menu open/close, updates aria-expanded, animates 3 spans into ✕
bookForm submit
Validates postcode + service + date, shows loading state, fires toast, scrolls to #providers
tab click
Filters .svc-card elements by data-cat attribute, toggling .gone class
IntersectionObserver
Two observers: one for [data-aos] scroll animations, one triggers stat counters on #stats entry
animateCounter()
setInterval-based count-up with support for k-suffix, decimal (4.9★), and locale formatting

index.html — Modals & Tracking

4 features
openModal() / closeModal()
Provider apply modal: toggles .open class, locks body scroll, focuses first input. Escape key + overlay click close
providerForm submit
Validates required fields, shows error borders, submits with simulated delay, resets + closes
trackOrder()
Looks up order by code in ORDERS object, calls buildResult() or buildNotFound(), animates panel in
buildResult()
Generates full tracking HTML: summary card, 5-step timeline, progress fill width, live status, ETA bar

admin.html — Auth & Navigation

5 features
doLogin()
Checks password === 'admin123', hides login screen, shows admin app, calls initApp()
doLogout()
Swaps visibility back to login screen, clears password field
nav(id)
Shows matching .panel, updates sidebar .on class, updates topbar title/subtitle, calls render function
setFilter()
Updates filters state object, toggles .on on tab buttons, re-renders the relevant table
toggleNotifs()
Opens/closes notification panel. Outside click listener auto-closes it

admin.html — Data Actions

12 features
renderOrders()
Filters ORDERS array by status + search query, builds full HTML table, updates row count
openOrderDetail()
Finds order, populates detail modal fields, sets status dropdown, opens modal

Mutates order.status in ORDERS array, closes modal, re-renders table + dashboard widget
approveProvider()
Sets provider.status = 'active', re-renders provider table, updates pending badge count
suspendProvider()
Sets provider.status = 'suspended', re-renders, fires notify toast

Updates provider.commission from inline input change event, fires notify
renderRevenueChart()
Generates SVG polyline chart: calculates x/y coords from data array, creates path + area fill + dots
blockCustomer() / unblockCustomer()
Mutates customer.status, re-renders customer table
toggleService()
Updates service.active boolean from toggle checkbox change
updateSvcPrice()
Updates service.price from inline price input, fires confirmation toast

Technology

Tech Stack

100% native web technologies — no frameworks, no build tools, no dependencies.

📄
HTML5 Semantic
nav, main, section, article, footer, header, aside. ARIA roles, labels, aria-live regions for accessibility. WCAG-compliant focus states.
🎨
CSS3 Advanced
Custom properties (CSS variables), CSS Grid, Flexbox, @keyframes animations, IntersectionObserver-triggered transitions, CSS pseudo-elements, repeating-linear-gradient textures, backdrop-filter.
⚡
Vanilla JavaScript
ES6+: const/let, arrow functions, array methods (filter, map, find, reduce), template literals, IntersectionObserver API, requestAnimationFrame, dynamic SVG generation. Zero libraries.
🔤
Google Fonts
Playfair Display (headings — elegant serif), DM Sans (body — clean geometric sans-serif), JetBrains Mono (code/IDs on this page). Loaded via @import.
📊
SVG Charts
Revenue line charts built as inline SVG: path (area fill) + polyline + circle dots. All coordinates calculated dynamically from data arrays in JavaScript. No Chart.js or D3.
📱
Responsive Design
Mobile-first CSS. Breakpoints at 480px, 600px, 768px, 860px, 1100px. CSS Grid auto-reflows from 3-col → 2-col → 1-col. Sidebar collapses to overlay on mobile.
♿
Accessibility
aria-label on all icons and buttons, role="navigation", role="dialog", aria-modal, aria-live="polite" on toasts and tracking results, focus management in modals, skip-to-content focus styles.
🚀
Performance
No external JS bundles. Fonts preloaded via Google CDN. IntersectionObserver for lazy animation triggering. Passive scroll listeners. CSS transitions use transform (GPU-composited).
